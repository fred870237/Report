I. 開場與引言
「大家好，今天我們要一起認識 Android 系統開發的基礎。Android 是全球最廣泛使用的行動作業系統，支援各種裝置，從手機到智慧電視、車載系統。學會 Android 開發，不僅能打造 App，更能理解系統背後的運作。」
II. Android 系統架構概覽
「Android 系統分為五層：
•	Linux Kernel：提供核心功能，如記憶體管理、驅動程式。
•	HAL：硬體抽象層，讓不同硬體能透過統一介面被系統使用。
•	Runtime：包含 ART/Dalvik，負責執行 Java/Kotlin 程式碼。
•	Framework：提供開發者 API，例如 Activity 管理、通知系統。
•	Application：最上層就是我們開發的 App。」
III. 開發環境與工具
「Android Studio 是官方整合開發環境，支援 Kotlin 與 Java。專案透過 Gradle 管理依賴與編譯。開發時可使用模擬器快速測試，也能連接真機。Debug 工具如 Logcat 可以輸出訊息，Profiler 則能分析效能。」
IV. 基礎開發流程
「建立第一個專案時，Android Studio 會生成一個 MainActivity。Activity 是 App 的主要入口，擁有生命週期：onCreate、onStart、onResume、onPause、onStop、onDestroy。 UI 使用 XML 定義，例如 Button、TextView，並透過程式碼綁定事件。資源檔案集中管理，方便多語系與不同裝置支援。」
V. 核心概念與元件
「Android 的四大元件：
•	Activity：畫面與互動。
•	Service：背景執行，例如音樂播放。
•	Broadcast Receiver：接收系統或 App 廣播事件。
•	Content Provider：跨 App 資料共享。
透過 Intent 可以在元件間傳遞訊息。資料儲存方式有 SharedPreferences、SQLite、Room。權限管理則需在 Manifest 宣告，並在執行時向使用者請求。」
VI. 總結與延伸方向
「今天我們快速走過 Android 系統開發的基礎：系統架構、開發工具、基本流程與核心元件。未來可以深入學習 Jetpack 套件、Kotlin 語言特性，以及 MVVM 架構設計。最重要的是持續練習，從簡單 App 開始，逐步累積經驗。」

🧱 Android 系統架構五層解說（中英對照）
層級	中文名稱	英文名稱	簡短解說（中英對照）
1（底層）	Linux 核心	Linux Kernel	提供記憶體管理、程序排程、驅動支援等底層功能。
Provides low-level functions like memory management, process scheduling, and driver support.
2	硬體抽象層	HAL (Hardware Abstraction Layer)	將硬體功能封裝成統一介面，供上層使用。
Encapsulates hardware functions into unified interfaces for upper layers.
3	Android 執行環境	Android Runtime (ART/Dalvik)	負責執行應用程式的 Java/Kotlin 程式碼。
Executes Java/Kotlin code for applications.
4	系統框架層	Framework Layer	提供開發者 API，如 Activity、Service、通知等系統服務。
Provides developer APIs like Activity, Service, and system services.
5（頂層）	應用程式層	Application Layer	使用者安裝與操作的 App 所在層級。
Layer where user-installed and operated apps reside.


 
📋 單選題題庫（10 題）
1. Android 系統的最底層是什麼？
A. Application Layer B. Framework Layer C. Linux Kernel D. Android Runtime 答案：C. Linux Kernel
2. Android 的硬體抽象層 (HAL) 主要作用是什麼？
A. 提供 UI 設計工具 B. 將硬體功能封裝成統一介面 C. 管理應用程式生命週期 D. 儲存使用者資料 答案：B. 將硬體功能封裝成統一介面
3. Android Runtime (ART) 的主要功能是？
A. 提供系統通知服務 B. 執行 Java/Kotlin 程式碼 C. 管理資源檔案 D. 控制硬體驅動程式 答案：B. 執行 Java/Kotlin 程式碼
4. Android 開發官方最推薦哪種語言來開發？
A. Java B. Kotlin C. C++ D. Python 答案：B. Kotlin
5. Android Studio 中用來管理依賴與編譯的工具是？
A. Maven B. Gradle C. Ant D. Makefile 答案：B. Gradle
6. 下列哪一個不是 Android 的四大元件？
A. Activity B. Service C. Broadcast Receiver D. Fragment 答案：D. Fragment （Fragment 是輔助 UI 元件，不屬於四大核心元件）
7. Activity 的生命週期中，哪個方法在 Activity 建立時最先被呼叫？
A. onStart B. onResume C. onCreate D. onPause 答案：C. onCreate
8. Android 中用來在元件間傳遞訊息的機制是？
A. Intent B. Bundle C. Service D. Content Provider 答案：A. Intent
9. Android 中用來儲存少量設定資料的方式是？
A. SQLite B. SharedPreferences C. Room Database D. Content Provider 答案：B. SharedPreferences
10. 在 Android 中，權限需要在哪裡宣告？
A. build.gradle B. AndroidManifest.xml C. strings.xml D. styles.xml 答案：B. AndroidManifest.xml
